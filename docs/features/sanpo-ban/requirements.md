# 要件定義書: 散歩判（sanpo-ban）

## 1. 概要

**散歩判**は、個人向けのダイエット情報ダッシュボードである。食事（カロリー・PFC）、体重、歩数（iPhone 自動同期）、散歩記録、トレッドミル、筋トレを一か所に集約し、今日の摂取余力と運動状況を把握できるようにする。

家計簿型アプリのような毎食の詳細入力負荷は避け、定番プリセット・最小限の手入力・（将来）バーコード読取でガチダイエットに必要な数値を追える設計とする。認証・認可は行わず、開発者本人が自宅 LAN 内で利用する。

## 2. 背景・目的

### 背景

- ダイエットを本気で行いたいが、記録をしていない（入力がだるい）
- 散歩はダイエット目的と、街での発見の楽しみの両方がある
- 食事で「食べていいか」迷うことがある
- ジムではトレッドミル・筋トレも行う
- 既存の汎用アプリ（家計簿・単機能歩数アプリ）は、情報が分散する

### 目的

- ダイエットに必要な情報を単一サービスに集約する
- 歩数は iPhone から自動取り込みし、手入力負荷を減らす
- PC 上でプロトタイプを完成させ、のちに Raspberry Pi（USB SSD）へ移行できるようにする

## 3. スコープ

### 3.1 対象（含む）

- Web ブラウザ UI（スマホ優先）
- REST API + MySQL によるデータ永続化
- 初期設定（身体情報・目標 kcal / PFC）
- 今日ダッシュボード（摂取・消費・残り kcal / PFC）
- 食事記録（プリセット・手入力）
- 体重記録（アプリ内入力 + iPhone 同期）
- 歩数の自動同期（iPhone ショートカット → API）
- 散歩セッション記録（1 タップ + 任意の発見メモ）
- トレッドミル記録（手入力）
- 筋トレ記録（クイック：種目テンプレ + 分数）
- 週次サマリー（摂取・歩数・体重・運動回数）
- PC ローカル開発環境（Docker Compose 想定）

### 3.2 対象外（含まない）

- ユーザー登録・ログイン・OAuth・外部 IdP
- 会社業務・GitHub / Cursor 連携
- クラウド SaaS ホスティング（初期リリース）
- Fitbit / Garmin 等クラウド API 連携
- Instagram / Twitter 自動取込
- ゲーム関連機能
- Phase 1 でのバーコード読取・Open Food Facts 連携（Phase 2）
- Phase 1 での筋トレ詳細（セット・重量）（Phase 2）
- Phase 1 での食べる判 ○△×（Phase 2）
- Raspberry Pi 本番デプロイ自動化（Phase 1 は PC 動作を優先。Pi 手順は backlog）

## 8. 制約・前提条件

### 技術的制約

- DB は MySQL 8.x を使用する
- 進め方 A: Phase 1 の完成・検証は PC ローカルで行う
- iPhone 歩数・体重同期は iOS ショートカットによる HTTP POST に依存する
- Open Food Facts は Phase 2。Phase 1 では外部 API 依存なし

### ビジネス制約

- 個人利用のみ。マルチテナント・課金・ストア公開は対象外

### 前提条件

- 利用者は iPhone を持ち、散歩時に携行する
- 利用者は自宅 Wi-Fi 上で Pi または PC にアクセスできる
- 利用者が iOS ショートカットを設定できる（設定手順は README / 運用ドキュメントで提供）

## 10. 用語定義

| 用語 | 定義 |
|------|------|
| PFC | タンパク質（Protein）・脂質（Fat）・炭水化物（Carbohydrate） |
| プリセット | 利用者が事前登録した定番食品（名称・kcal・P/F/C） |
| 散歩セッション | 「散歩した」操作で記録されるイベント。発見メモを任意で付与 |
| Health 同期 | iPhone ショートカットが API へ歩数・体重を POST すること |
| 推定消費 kcal | 歩数・トレッドミル・筋トレから算出した運動消費カロリー |
| Open Food Facts | OSS の食品データベース。バーコードから栄養成分を取得（Phase 2） |
