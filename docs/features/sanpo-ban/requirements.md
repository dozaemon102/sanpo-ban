# 要件定義書: 健康管理（kenko-kanri）

## 1. 概要

**健康管理**（旧称: 散歩判 / sanpo-ban）は、個人向けのダイエット・体組成ダッシュボードである。食事（カロリー・PFC）、体重・体組成（BMI / LBM / 体脂肪率）、歩数（iPhone 自動同期）、トレッドミル、筋トレを一か所に集約し、**カロリー収支**の赤字（マイナス）を可視化してダイエット継続を支援する。

**v2（2026-06）:** Open Food Facts 連携によるバーコード食事記録。  
**v3（2026-06）:** UI 刷新、収支モデル変更（Katch–McArdle + NEAT + TEF）、アプリ全面リネーム、散歩記録廃止、週タブをカード履歴に統合。

認証・認可は行わず、開発者本人が LAN / Tailscale 内で利用する。

## 2. 背景・目的

### 背景

- ダイエットを本気で行いたいが、記録をしていない（入力がだるい）
- 散歩はダイエット目的と、街での発見の楽しみの両方がある
- 食事で「食べていいか」迷うことがある
- ジムではトレッドミル・筋トレも行う
- 既存の汎用アプリ（家計簿・単機能歩数アプリ）は、情報が分散する

### 目的

- ダイエットに必要な情報を単一サービスに集約する
- **カロリー収支の赤字を可視化**し、継続しやすくする
- 歩数・体組成は iPhone から自動取り込みし、手入力負荷を減らす
- Raspberry Pi（USB SSD）上で常時稼働する

## 3. スコープ

### 3.1 対象（含む）

- Web ブラウザ UI（スマホ優先・PWA）
- REST API + MySQL によるデータ永続化
- 設定（身体情報・NEAT・TEF 率）
- TOP 収支ダッシュボード（v3）
- 食事記録（Myセット・手入力・バーコード）
- 体重・体組成記録（手入力 + iPhone Health 同期）
- 歩数の自動同期（iPhone ショートカット → API）
- トレッドミル・筋トレ記録
- カード別履歴（日 / 週 / 月 / 年）（v3）
- PC ローカル開発環境 / Pi 本番

### 3.2 対象外（含まない）

- ユーザー登録・ログイン・OAuth・外部 IdP
- 会社業務・GitHub / Cursor 連携
- クラウド SaaS ホスティング（初期リリース）
- Fitbit / Garmin 等クラウド API 連携
- Instagram / Twitter 自動取込
- ゲーム関連機能
- 食べる判 ○△×（FR-016）、筋トレ詳細（FR-032）、旅モード（FR-034/035）
- Raspberry Pi 本番デプロイ自動化（Pi 手順は backlog）
- **v3 で廃止:** 散歩セッション記録（FR-024〜026）、週タブ単独画面（FR-033 UI）、TOP の目標 kcal / PFC 残量（FR-002/003/005）、PFC 目標・残量表示（食事タブ）

## 8. 制約・前提条件

### 技術的制約

- DB は MySQL 8.x を使用する
- 進め方 A: Phase 1 の完成・検証は PC ローカルで行う
- iPhone 歩数・体重同期は iOS ショートカットによる HTTP POST に依存する
- **v2:** Open Food Facts API を Pi（バックエンド）から呼び出す。商品バーコードは EAN-13 / JAN を想定（詳細は基本設計）
- **v3:** 収支モデル・体組成（BMI/LBM/体脂肪率）Health 同期拡張済み
- **v3:** プロジェクト slug を `kenko-kanri` にリネーム（DB 名は移行方針を基本設計で決定）

### ビジネス制約

- 個人利用のみ。マルチテナント・課金・ストア公開は対象外

### 前提条件

- 利用者は iPhone を持ち、散歩時に携行する
- トレッドミル中はスマホを携行しない（歩数にジム走行分は含まれず、二重計上は発生しない）
- 利用者は自宅 Wi-Fi または Tailscale 上で Pi にアクセスできる
- 利用者が iOS ショートカットを設定できる（設定手順は README / 運用ドキュメントで提供）

## 10. 用語定義

| 用語 | 定義 |
|------|------|
| PFC | タンパク質（Protein）・脂質（Fat）・炭水化物（Carbohydrate） |
| プリセット | 利用者が事前登録した定番食品（名称・kcal・P/F/C） |
| 散歩セッション | 「散歩した」操作で記録されるイベント。発見メモを任意で付与 |
| Health 同期 | iPhone ショートカットが API へ歩数・体重を POST すること |
| カロリー収支 | 摂取 − 基礎代謝 − NEAT − 運動 − TEF。マイナス＝赤字＝痩せ方向 |
| 基礎代謝（v3） | Katch–McArdle 式。LBM 必須（最新値） |
| NEAT | Non-Exercise Activity Thermogenesis。v3 初期 200 kcal 固定（設定可） |
| TEF | Thermic Effect of Food。v3 初期は摂取 kcal の 10%（設定可） |
| Myセット | 旧プリセット。定番食事・運動テンプレの登録セット |
| 健康管理 | v3 アプリ表示名。slug: kenko-kanri |
| Open Food Facts | OSS の食品データベース。バーコードから栄養成分を取得（v2 で利用） |
| バーコード読取 | スマホブラウザのカメラで JAN/EAN を読み取り、サーバーが OFF を参照する |
