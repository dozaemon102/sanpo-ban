# 事前調査: 散歩判（sanpo-ban）

> **調査方法:** 会話イントークで十分な論点が確認済みのため、Web 調査は省略し既知情報を整理した。

## 1. 調査目的

個人向けダイエットダッシュボードの実現可能性（iPhone 歩数自動化、食事 PFC、MySQL、Pi ホスティング）を確認する。

## 2. Open Food Facts（OFF）

| 項目 | 内容 |
|------|------|
| ライセンス | データ: ODbL。ソフトウェア: OSS（GitHub: openfoodfacts） |
| API | 無料。バーコード → 栄養成分（kcal, P, F, C 等） |
| 日本向け | 海外製品・一部国内品は充実。未登録商品はプリセット・手入力が必要 |
| 本プロジェクトでの位置 | Phase 2（バーコード）。Phase 1 はプリセット + 手入力 |

## 3. iPhone 歩数・体重の自動同期

| 項目 | 内容 |
|------|------|
| 方式 | iOS ショートカット → 自宅 LAN 上の API へ POST |
| データ源 | ヘルスケアアプリ（歩数は日次累計、体重は任意） |
| 認証 | Apple Developer / OAuth 不要 |
| 前提 | 散歩時に iPhone を携行（ユーザー確認済み） |
| 制約 | ブラウザのみの Web アプリから HealthKit へ直接アクセス不可 |

## 4. トレッドミル・筋トレ

| 種別 | 記録方式 | 備考 |
|------|----------|------|
| 外散歩 | 歩数自動 + 「散歩した」ボタン | 発見メモは任意 |
| トレッドミル | 手入力（分数、任意で速度・傾斜 or マシン kcal） | ジム中スマホ非携行 → 歩数二重計上なし |
| 筋トレ | クイック（種目テンプレ + 分数）を Phase 1 | 詳細ログは Phase 2 |

## 5. MySQL + Raspberry Pi

| 項目 | 内容 |
|------|------|
| DB | MySQL 8.x（ユーザー指定） |
| 開発 | PC 上 Docker Compose で MySQL + API |
| 本番 | Pi 上 MySQL、データディレクトリを USB SSD に配置 |
| SD 対策 | SSD 推奨（OS 起動 or データのみマウント）。書き込み頻度は歩数同期・食事ログ程度 |

## 6. 競合・差別化

| 既存カテゴリ | 散歩判との差 |
|-------------|-------------|
| 家計簿型ダイエットアプリ | 入力最小化、個人 LAN、認証なし |
| 歩数アプリ | ダイエット全体（PFC・筋トレ・体重）を集約 |
| MyFitnessPal 等 | クラウド・認証必須。本件は自己ホスト |

## 7. リスク・未調査領域

| リスク | 対策案 |
|--------|--------|
| OFF 日本商品の欠落 | ユーザー定義プリセット |
| 消費 kcal 推定精度 | 体重連動 MET、マシン表示 kcal の直接入力を許容 |
| LAN 外からのアクセス | スコープ外（VPN 等は将来） |
